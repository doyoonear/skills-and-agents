# 문서 작성 가이드

## 문서 작성 시점

아래와 같은 경우, 사용자가 요청하지 않아도 항상 skills 디렉토리에 새로운 문서를 작성하거나, 기존 문서를 업데이트합니다.

1. 새로운 도구를 사용하기 위해 환경을 세팅한 경우
2. 기존 도구의 환경 세팅을 변경한 경우
3. 3개의 메세지 이상의 연관된 트러블슈팅 작업을 통해 문제를 해결한 경우

## 문서의 작성 방향

- 해당 환경을 세팅한 목적, 용도, 워크플로우, 사용방법에 관한 md 형식의 문서를 제작합니다.
- 이후 다른 AI Agent가 참고하여 문제를 해결하거나 기능을 구현할 수 있도록 충분한 정보를 제공합니다.
- 문서를 제작할때는 api key, secret 등 민감한 정보가 포함되지않도록 하세요. 필요한 예시는 placeholder 로 대체하세요.
- 작성 후 문서에서 중복되는 내용이 없는지 재검토하여 간략하게 요약하세요. 한개 문서의 총 길이는 600 라인을 넘지 않도록하고, 가능하다면 간략하게 500 라인 이내로 유지하세요.
- 필요한 경우, 연관한 문서를 묶기 위한 디렉토리를 생성하여 연관 문서들을 관리하세요.
- 트러블슈팅 문서는 실제 세션에서 문제를 해결한 과정을 바탕으로만 작성하고, 가상의 시나리오를 바탕으로 작성하지 않습니다.
