# MECE 분석 & 구조적 글쓰기

맥킨지의 MECE 원칙과 바바라 민토의 피라미드 원칙을 결합하여, 어떤 주제든 겹침 없이(ME) 빠짐없이(CE) 분류·분석한 뒤 체계적인 글로 작성하는 스킬입니다.

## 트리거 조건

다음 요청 시 이 스킬을 사용하세요:

- "MECE로 정리해줘", "MECE 분석해줘"
- "구조적으로 글 써줘", "체계적으로 정리해줘"
- "맥킨지 프레임워크로 분석해줘"
- "이슈 트리 만들어줘"
- "빠짐없이 분류해줘", "겹치지 않게 나눠줘"
- "피라미드 원칙으로 정리해줘"

한글 트리거: "MECE", "미시", "구조적 분석", "체계적 정리", "프레임워크 분석", "이슈트리"

## 핵심 원칙

### MECE란?

| 원칙 | 의미 | 검증 질문 |
|------|------|-----------|
| **Mutually Exclusive (상호 배제)** | 각 분류 항목 간에 중복이 없다 | "두 항목에 동시에 해당되는 경우가 있는가?" |
| **Collectively Exhaustive (전체 포괄)** | 모든 항목의 합이 전체를 빠짐없이 커버한다 | "누락된 경우의 수가 있는가?" |

### 피라미드 원칙 (Barbara Minto)

```
[핵심 메시지 / 결론]          ← 답을 먼저
        ↓
[MECE한 3~4개의 핵심 근거]    ← 겹침 없고 빠짐없는 지지 논거
        ↓
[각 근거를 뒷받침하는 데이터/사실]
```

## 워크플로우

사용자가 MECE 분석 및 글쓰기를 요청하면 다음 5단계를 순서대로 수행합니다.

### Step 1: 주제 파악 & 프레임워크 추천

사용자에게 다음을 확인합니다:

1. **주제**: 분석/글쓰기 대상 (예: "SaaS 시장 진입 전략", "이직 결정 요인")
2. **목적**: 분석만 / 글쓰기까지 / 의사결정 지원
3. **대상 독자**: 누구를 위한 글인가
4. **분석 깊이**: 2단계(기본) / 3단계(심층)

확인 후, 주제에 가장 적합한 **프레임워크를 2~3개 추천**하고 사용자가 선택하도록 합니다.
자유 분류가 더 적합한 주제라면 "커스텀 MECE 분류"를 제안합니다.

**추천 시 고려사항:**

| 주제 유형 | 1순위 프레임워크 | 2순위 프레임워크 |
|-----------|-----------------|-----------------|
| 시장/산업 분석 | Porter's 5 Forces | PESTEL |
| 마케팅 전략 | STP + 4P | 3C |
| 비즈니스 전략 | 3C | SWOT |
| 수익성/재무 분석 | 수익성 분해 공식 | 가치사슬 |
| 조직/내부 진단 | McKinsey 7S | SWOT |
| 제품 포트폴리오 | BCG Matrix | Ansoff Matrix |
| 성장 전략 | Ansoff Matrix | 3C |
| 거시환경 분석 | PESTEL | Porter's 5 Forces |
| 고객 분석 | STP | Jobs-to-be-Done |
| 일반 주제/비비즈니스 | 커스텀 MECE 분류 | 프로세스 기반 분류 |

### Step 2: MECE 이슈 트리 구성

선택된 프레임워크 또는 커스텀 분류를 기반으로 이슈 트리를 구성합니다.

**분류 방법론 5가지 중 주제에 맞는 것을 선택:**

| 방법 | 설명 | 적합한 경우 |
|------|------|-------------|
| **수식 기반** | 수학적 등식으로 분해 (자동 MECE 보장) | 재무/수익성 분석 |
| **세그먼트 기반** | 고객군별, 지역별, 채널별 분류 | 시장 세분화, 고객 분석 |
| **프로세스 기반** | 시간적/논리적 순서로 단계 분류 | 고객 여정, 가치사슬, 업무 프로세스 |
| **대립 축 기반** | 내부/외부, 단기/장기 등 2×2 매트릭스 | SWOT, BCG, 비교 분석 |
| **프레임워크 기반** | 검증된 프레임워크 직접 적용 | 전략 분석 전반 |

**2단계 구조 (기본):**

```
[핵심 주제]
├── 대분류 A
│   ├── 소분류 A-1
│   └── 소분류 A-2
├── 대분류 B
│   ├── 소분류 B-1
│   └── 소분류 B-2
└── 대분류 C
    ├── 소분류 C-1
    └── 소분류 C-2
```

**3단계 구조 (심층 — 사용자 요청 시):**

```
[핵심 주제]
├── 대분류 A
│   ├── 중분류 A-1
│   │   ├── 소분류 A-1-a
│   │   └── 소분류 A-1-b
│   └── 중분류 A-2
│       ├── 소분류 A-2-a
│       └── 소분류 A-2-b
├── 대분류 B
│   └── ...
└── 대분류 C
    └── ...
```

**분류 원칙:**
- 대분류는 3~5개가 적정 (7개 초과 시 상위 개념으로 재묶기)
- 소분류는 각 대분류 당 2~4개가 적정
- 동일한 논리적 수준(Logical Level)을 유지

### Step 3: MECE 검증

이슈 트리 구성 후 반드시 다음 검증을 수행합니다.

**ME(상호 배제) 검증:**

- [ ] 각 항목을 독립적으로 변경할 수 있는가?
- [ ] 하나의 데이터가 두 개 이상의 카테고리에 동시에 해당되지 않는가?
- [ ] 카테고리들이 동일한 논리적 수준에 있는가?
- [ ] 각 카테고리의 정의와 경계가 명확한가?

**CE(전체 포괄) 검증:**

- [ ] "이 외에 또 다른 경우가 없는가?"를 반복 질문했는가?
- [ ] 엣지 케이스(경계 사례)를 포함했는가?
- [ ] 모든 하위 항목의 합이 상위 항목을 100% 설명하는가?
- [ ] 분석 후 "원인을 못 찾겠다"는 느낌이 들지 않는가?

**일반적인 MECE 위반 패턴 (이것들을 피할 것):**

| 위반 유형 | 잘못된 예시 | 문제점 |
|-----------|-------------|--------|
| 겹치는 카테고리 | "빨간 차" vs "스포츠카" | 빨간 스포츠카는 양쪽에 해당 |
| 불완전한 구분 | "50세 미만" vs "50세 초과" | 정확히 50세인 경우 누락 |
| 복합 속성 혼용 | "어린이, 청소년, 30세 미만" | "30세 미만"이 앞 항목과 중복 |
| 다른 레벨 혼용 | "마케팅, 영업, 고객 획득" | 고객 획득은 하위 개념 |
| 편향된 포괄 | 매출 분석에서 "가격 + 비용"만 | 판매량, 부가 수익 등 누락 |

**검증 실패 시:** 이슈 트리를 수정하고 Step 3을 다시 수행합니다.

### Step 4: 분석 & 글 작성

검증을 통과한 MECE 이슈 트리를 바탕으로 글을 작성합니다.

**글 구성 템플릿:**

```markdown
# [주제 제목]

## 개요
- 배경/맥락 (Situation)
- 핵심 문제/질문 (Complication → Question)
- 이 글의 핵심 메시지 (Answer — 결론 먼저)

## 분석 프레임워크
- 사용한 프레임워크: [프레임워크명]
- 분류 방법: [분류 방법론]
- 분석 깊이: [2단계/3단계]

## MECE 분류 트리
[트리 구조 시각화]

## 상세 분석

### 1. [대분류 A]

#### 1-1. [소분류 A-1]
- 핵심 포인트
- 근거/데이터
- 시사점

#### 1-2. [소분류 A-2]
- 핵심 포인트
- 근거/데이터
- 시사점

### 2. [대분류 B]
...

### 3. [대분류 C]
...

## 분류 간 관계성 & 종합 시사점
- 대분류 간 상호작용/영향 관계
- 우선순위 또는 중요도 평가
- 핵심 인사이트 종합

## 결론 & 제언
- 핵심 메시지 재확인
- 구체적 제언/액션 아이템
- 향후 과제
```

**글쓰기 원칙:**
- 피라미드 원칙에 따라 결론을 먼저 제시(Top-down)
- 각 분류 항목은 "주장 → 근거 → 시사점" 순서로 서술
- 분류 간 전환 시 자연스러운 연결문 사용
- 구체적 데이터, 예시, 사례를 활용하여 설득력 강화

### Step 5: 파일 저장 & 최종 검토

**파일명 규칙:** `mece-[주제 키워드]-[YYYYMMDD].md`
- 예: `mece-saas-market-entry-20260226.md`
- 예: `mece-career-change-factors-20260226.md`

**저장 위치:** 현재 작업 디렉토리

**최종 검토 항목:**
- [ ] MECE 원칙 준수 여부 최종 확인
- [ ] 피라미드 구조 (결론 먼저) 유지
- [ ] 각 섹션의 논리적 흐름
- [ ] 누락된 관점이나 분류가 없는지 확인

저장 후 사용자에게 다음을 보고합니다:
- 파일 저장 경로
- 사용한 프레임워크
- MECE 분류 트리 요약
- 글의 핵심 메시지

## 내장 프레임워크 레퍼런스

사용자가 프레임워크를 선택할 때 참고할 수 있도록 주요 프레임워크의 구조를 내장합니다.

### 3C Framework
**적용:** 비즈니스 전략 수립, 시장 진입 전략

| 구성 요소 | 분석 항목 |
|-----------|-----------|
| **Company (자사)** | 강점, 약점, 핵심역량, 재무상태 |
| **Customer (고객)** | 니즈, 세분화, 구매행동, 트렌드 |
| **Competitor (경쟁사)** | 경쟁 구도, 차별화 포인트, 시장점유율 |

### 4P Framework (Marketing Mix)
**적용:** 마케팅 전략 설계

| 구성 요소 | 분석 항목 |
|-----------|-----------|
| **Product (제품)** | 기능, 품질, 디자인, 브랜드, 패키징 |
| **Price (가격)** | 가격 전략, 할인, 지불 조건 |
| **Place (유통)** | 유통 채널, 물류, 접근성 |
| **Promotion (촉진)** | 광고, PR, 판촉, 디지털 마케팅 |

### STP Framework
**적용:** 타겟 마케팅, 브랜드 포지셔닝

| 단계 | 핵심 활동 |
|------|-----------|
| **Segmentation (시장 세분화)** | 인구통계, 심리통계, 행동 기반 분류 |
| **Targeting (타겟 선정)** | 매력도 평가, 자사 역량 매칭 |
| **Positioning (포지셔닝)** | USP 정의, 포지셔닝 맵 작성 |

### SWOT Analysis
**적용:** 전략적 상황 진단

| 구분 | 내부 요인 | 외부 요인 |
|------|-----------|-----------|
| **긍정적** | Strengths (강점) | Opportunities (기회) |
| **부정적** | Weaknesses (약점) | Threats (위협) |

### Porter's 5 Forces
**적용:** 산업 매력도 분석, 경쟁 전략 수립

| 힘 | 분석 질문 |
|----|-----------|
| **신규 진입자 위협** | 진입장벽이 낮아 새 경쟁자가 쉽게 들어올 수 있는가? |
| **공급자 협상력** | 공급자가 가격/조건을 좌우할 수 있는가? |
| **구매자 협상력** | 고객이 가격 인하나 조건 개선을 강요할 수 있는가? |
| **대체재 위협** | 제품을 대체할 수 있는 것이 있는가? |
| **기존 경쟁 강도** | 업계 내 경쟁이 얼마나 치열한가? |

### Value Chain (가치사슬)
**적용:** 원가 분석, 경쟁우위 원천 파악

**1차 활동:** 내부 물류 → 운영/생산 → 외부 물류 → 마케팅 & 영업 → 서비스
**지원 활동:** 인프라, 인적자원관리, 기술개발, 조달

### McKinsey 7S Framework
**적용:** 조직 변화 관리, 내부 정렬 분석

| 구분 | 요소 |
|------|------|
| **Hard S** | Strategy(전략), Structure(구조), Systems(시스템) |
| **Soft S** | Shared Values(공유 가치), Skills(역량), Staff(인재), Style(리더십) |

### BCG Matrix
**적용:** 포트폴리오 관리, 투자 우선순위

| 구분 | 시장성장률 낮음 | 시장성장률 높음 |
|------|---------------|---------------|
| **점유율 높음** | Cash Cow (수확) | Star (투자 유지) |
| **점유율 낮음** | Dog (철수/매각) | Question Mark (선택적 투자) |

### PESTEL
**적용:** 거시환경 분석

| 요인 | 분석 항목 |
|------|-----------|
| **Political (정치)** | 규제, 정부 정책, 정치적 안정성 |
| **Economic (경제)** | 경기 순환, 금리, 환율, 인플레이션 |
| **Social (사회)** | 인구통계, 라이프스타일, 문화적 트렌드 |
| **Technological (기술)** | 기술 혁신, R&D, 자동화, 디지털 전환 |
| **Environmental (환경)** | 기후변화, 지속가능성, 환경 규제 |
| **Legal (법적)** | 노동법, 소비자보호법, 지적재산권 |

### Ansoff Matrix
**적용:** 성장 전략

| 구분 | 기존 제품 | 신규 제품 |
|------|-----------|-----------|
| **기존 시장** | 시장 침투 | 제품 개발 |
| **신규 시장** | 시장 개발 | 다각화 |

### 수익성 분해 공식
**적용:** 재무/수익성 분석 (수식 기반 — 자동 MECE 보장)

```
수익(Profit) = 매출(Revenue) - 비용(Cost)
├── 매출 = 가격(Price) × 판매량(Volume)
│   ├── 가격: 단가, 할인율, 프리미엄
│   └── 판매량: 신규 고객 + 기존 고객 유지
│       ├── 신규: 유입 × 전환율
│       └── 기존: 재구매율 × 구매 빈도
└── 비용 = 고정비(Fixed) + 변동비(Variable)
    ├── 고정비: 인건비, 임대료, 감가상각
    └── 변동비: 원재료, 물류비, 수수료
```

### Jobs-to-be-Done
**적용:** 고객 니즈 분석

| 니즈 유형 | 설명 | 예시 |
|-----------|------|------|
| **기능적 니즈** | 실질적 문제 해결 | "빠르게 이동하고 싶다" |
| **사회적 니즈** | 타인에게 보이는 이미지 | "성공한 사람으로 보이고 싶다" |
| **감정적 니즈** | 내면의 감정 충족 | "안전하다고 느끼고 싶다" |

## 커스텀 MECE 분류 가이드

내장 프레임워크가 적합하지 않은 주제(일상, 자기계발, 취미 등)에는 커스텀 분류를 생성합니다.

**커스텀 분류 절차:**

1. **주제의 전체 범위 정의**: "이 주제가 포괄하는 모든 것은 무엇인가?"
2. **분류 축(Dimension) 선택**: 시간, 대상, 원인, 방법, 장소 등
3. **대분류 도출**: 선택한 축을 기준으로 3~5개 대분류 생성
4. **ME 검증**: 각 대분류가 겹치지 않는지 확인
5. **CE 검증**: 모든 대분류의 합이 전체를 커버하는지 확인
6. **소분류 전개**: 각 대분류를 2~4개 소분류로 세분화
7. **재검증**: 소분류 수준에서 ME/CE 재확인

**자주 사용되는 분류 축:**

| 분류 축 | 예시 |
|---------|------|
| 시간 | 과거/현재/미래, 단기/중기/장기 |
| 대상 | 개인/팀/조직, 내부/외부 |
| 원인 | 구조적/운영적/전략적 |
| 방법 | 온라인/오프라인, 정량/정성 |
| 수준 | 전략/전술/실행 |
| 범위 | 로컬/내셔널/글로벌 |
