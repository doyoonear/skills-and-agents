# 핸드오프 템플릿

아래 템플릿의 `{placeholder}`를 실제 값으로 대체하여 핸드오프 문서를 작성한다.

```markdown
# {주제 타이틀}

| 항목 | 내용 |
|------|------|
| 날짜 | {YYYY-MM-DD} |
| 프로젝트 | {프로젝트명} |
| 브랜치 | {브랜치명} |
| 상태 | {진행중 / 일시중단 / 블로킹} |

## 배경

{이 작업이 왜 필요한지, 어떤 맥락에서 시작되었는지 간략히 설명}

## 완료 사항

- [x] {완료된 작업 1}
- [x] {완료된 작업 2}

## 진행 중

- {현재 진행 중인 작업과 현재 상태}

## 남은 TODO

- [ ] {해야 할 작업 1}
- [ ] {해야 할 작업 2}
- [ ] {해야 할 작업 3}

## 주요 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| {파일 경로} | {변경 설명} |

## 기술적 결정사항

| 결정 | 이유 |
|------|------|
| {결정 내용} | {왜 이렇게 결정했는지} |

## 주의사항

- {알려진 이슈나 주의할 점}
- {워크어라운드가 필요한 부분}

## 참고

- {관련 PR, 이슈, 문서 링크 등}
```
