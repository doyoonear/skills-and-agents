# Handoff - 세션 간 작업 인수인계

현재 세션의 작업 상태를 `docs/handoff/` 폴더에 주제별 마크다운 문서로 정리하여, 다음 세션에서 이어받을 수 있게 한다.

## 모드 판별

`docs/handoff/` 폴더의 존재 여부와 사용자 의도에 따라 모드를 판별한다.

### 생성 모드 (Create)

사용자가 현재 작업을 정리하고 싶을 때 실행한다.

### 이어받기 모드 (Resume)

`docs/handoff/` 폴더에 기존 문서가 있고, 사용자가 작업을 이어받으려 할 때 실행한다.

---

## 생성 모드 워크플로우

### 1. 주제명 확인

사용자에게 핸드오프 주제명을 질문한다. (파일명에 사용)

- 예: "레이아웃 리디자인", "노션 동기화 파이프라인"
- 파일명 형식: `{YYYY-MM-DD}-{주제-kebab-case}.md`

### 2. 자동 정보 수집

다음 git 명령어를 **병렬로** 실행하여 프로젝트 상태를 수집한다:

```bash
git branch --show-current
git status --short
git diff --stat
git log --oneline -10
```

### 3. 컨텍스트 분석

현재 대화 컨텍스트에서 다음 정보를 파악한다:

- 이번 세션에서 완료한 작업
- 현재 진행 중인 작업
- 남은 TODO 항목
- 기술적 결정사항과 그 이유
- 주의사항이나 알려진 이슈

### 4. 핸드오프 문서 작성

`docs/handoff/` 디렉토리에 문서를 작성한다. 디렉토리가 없으면 생성.

**템플릿:** `references/handoff-template.md` 참조

### 5. CLAUDE.md 연동 확인

프로젝트의 `CLAUDE.md`에 핸드오프 참조 지시가 없으면 추가한다:

```markdown
# 핸드오프

- 세션 시작 시 `docs/handoff/` 폴더가 존재하면 문서를 확인하고, 어떤 작업을 이어서 진행할지 사용자에게 질문합니다.
```

### 6. 완료 보고

- 생성된 핸드오프 파일 경로
- 문서에 포함된 TODO 개수
- 다음 세션에서 이어받는 방법 안내

---

## 이어받기 모드 워크플로우

### 1. 핸드오프 문서 목록 확인

```bash
ls docs/handoff/
```

### 2. 문서 읽기

모든 핸드오프 문서를 읽고 내용을 파악한다.

### 3. 사용자에게 질문

각 핸드오프 문서의 요약을 제시하고, 어떤 작업을 이어서 할지 사용자에게 질문한다.

- 문서가 1개면: 해당 문서 요약 후 이어서 진행할지 확인
- 문서가 여러 개면: 목록을 보여주고 선택하게 함

### 4. 선택된 작업 시작

사용자가 선택한 핸드오프 문서의 TODO를 기반으로 작업을 시작한다.

### 5. 완료된 핸드오프 정리

작업이 모두 완료된 핸드오프 문서는 사용자 확인 후 삭제하거나 `docs/handoff/done/`으로 이동한다.

---

## 주의사항

- 핸드오프 문서는 **커밋하지 않는다** (사용자가 원하면 별도로 커밋)
- `wrap` 스킬과 역할이 다름: `wrap`은 커밋/PR 생성, `handoff`는 세션 간 컨텍스트 전달
- 핸드오프 문서에는 민감 정보(API 키, 비밀번호 등)를 포함하지 않는다
